Lesson I

Selection Structures

Learn how to control program flow using conditional statements to make decisions based on specific conditions.

Lesson Overview

Selection structures, also known as decision-making statements, allow your programs to choose different paths of execution based on conditions. Just as we make decisions in everyday life—"If it's raining, I'll take an umbrella"—programs use selection structures to respond dynamically to different situations.

Real-World Context: Every application you use relies on selection structures. When you log into a website, the system checks if your password is correct. ATMs verify if you have sufficient funds before allowing withdrawals. Games check if a player's health reaches zero to trigger a game-over screen.

Learning Outcomes

By the end of this lesson, you will be able to:

  • Write programs using the if statement to execute code conditionally
  • Implement if...else statements to handle two alternative paths
  • Construct if...else if...else chains for multiple conditions
  • Apply switch statements for multi-way branching based on discrete values
  • Design nested decision structures for complex conditional logic

1. The if Statement

The if statement is the simplest form of selection structure. It executes a block of code only when a specified condition evaluates to true.

Syntax

Java Syntax 
if (condition) {
    // Code to execute if condition is true
}

The condition must be a boolean expression—an expression that evaluates to either true or false. Common comparison operators include:

Operator Description Example
== Equal to x == 5
!= Not equal to x != 5
> Greater than x > 5
< Less than x < 5
>= Greater than or equal to x >= 5
<= Less than or equal to x <= 5

Example: Checking Voting Eligibility

The following program checks if a person is eligible to vote based on their age:

Java 
public class VotingEligibility {
    public static void main(String[] args) {
        int age = 18;  // Variable to store the person's age
        
        // Check if age meets the voting requirement
        if (age >= 18) {
            System.out.println("You are eligible to vote.");
        }
        
        System.out.println("Program completed.");
    }
}
Output:
You are eligible to vote.
Program completed.

2. The if...else Statement

The if...else statement provides an alternative path when the condition is false. This ensures that one of two code blocks will always execute.

Syntax

Java Syntax 
if (condition) {
    // Code to execute if condition is true
} else {
    // Code to execute if condition is false
}

Example: Pass or Fail

This program determines whether a student passed or failed based on their score:

Java 
public class GradeChecker {
    public static void main(String[] args) {
        int score = 72;  // Student's exam score
        int passingScore = 60;  // Minimum passing score
        
        // Determine if student passed or failed
        if (score >= passingScore) {
            System.out.println("Congratulations! You passed.");
            System.out.println("Your score: " + score);
        } else {
            System.out.println("Sorry, you failed.");
            System.out.println("You need " + (passingScore - score) + " more points.");
        }
    }
}
Output:
Congratulations! You passed.
Your score: 72

Example: Even or Odd Number

Using the modulo operator to determine if a number is even or odd:

Java 
public class EvenOddChecker {
    public static void main(String[] args) {
        int number = 7;  // Number to check
        
        // Check if number is divisible by 2 with no remainder
        if (number % 2 == 0) {
            System.out.println(number + " is even.");
        } else {
            System.out.println(number + " is odd.");
        }
    }
}

3. The if...else if...else Statement

When you need to test multiple conditions sequentially, use the if...else if...else chain. The program evaluates each condition in order and executes the first block whose condition is true.

Syntax

Java Syntax 
if (condition1) {
    // Code for condition1
} else if (condition2) {
    // Code for condition2
} else if (condition3) {
    // Code for condition3
} else {
    // Code if no condition is true
}

Example: Letter Grade Assignment

This program converts a numeric score to a letter grade:

Java 
public class GradeConverter {
    public static void main(String[] args) {
        int score = 85;  // Student's score
        char grade;  // Variable to store the letter grade
        
        // Determine letter grade based on score range
        if (score >= 90) {
            grade = 'A';  // 90-100: Excellent
        } else if (score >= 80) {
            grade = 'B';  // 80-89: Good
        } else if (score >= 70) {
            grade = 'C';  // 70-79: Satisfactory
        } else if (score >= 60) {
            grade = 'D';  // 60-69: Passing
        } else {
            grade = 'F';  // Below 60: Failing
        }
        
        System.out.println("Score: " + score);
        System.out.println("Grade: " + grade);
    }
}
Output:
Score: 85
Grade: B
Important: The order of conditions matters. Conditions are evaluated from top to bottom, and only the first true condition's block is executed. Always order your conditions from most specific to least specific.

4. The switch Statement

The switch statement provides a cleaner alternative to long if...else if chains when testing a single variable against multiple discrete values. It's particularly useful for menu systems and categorical data.

Syntax

Java Syntax 
switch (expression) {
    case value1:
        // Code for value1
        break;
    case value2:
        // Code for value2
        break;
    case value3:
        // Code for value3
        break;
    default:
        // Code if no case matches
}

Example: Day of the Week

This program displays the day name based on a numeric value:

Java 
public class DayOfWeek {
    public static void main(String[] args) {
        int day = 3;  // Day number (1-7)
        String dayName;  // Variable to store day name
        
        // Match day number to day name
        switch (day) {
            case 1:
                dayName = "Monday";
                break;
            case 2:
                dayName = "Tuesday";
                break;
            case 3:
                dayName = "Wednesday";
                break;
            case 4:
                dayName = "Thursday";
                break;
            case 5:
                dayName = "Friday";
                break;
            case 6:
                dayName = "Saturday";
                break;
            case 7:
                dayName = "Sunday";
                break;
            default:
                dayName = "Invalid day";
        }
        
        System.out.println("Day " + day + ": " + dayName);
    }
}
Output: Day 3: Wednesday

Switch with String (Java 7+)

Since Java 7, you can use String values in switch statements:

Java 
public class MenuSelector {
    public static void main(String[] args) {
        String command = "save";  // User command
        
        switch (command.toLowerCase()) {
            case "new":
                System.out.println("Creating new file...");
                break;
            case "open":
                System.out.println("Opening file...");
                break;
            case "save":
                System.out.println("Saving file...");
                break;
            case "exit":
                System.out.println("Exiting program...");
                break;
            default:
                System.out.println("Unknown command.");
        }
    }
}
The break Statement: Each case should end with break to prevent "fall-through" to the next case. Without break, execution continues to subsequent cases until a break is encountered or the switch ends.

5. Nested Decision Structures

Nested decisions occur when one selection structure is placed inside another. This allows for complex multi-level decision making where subsequent conditions depend on previous ones.

Example: Loan Eligibility

A bank checks multiple criteria to determine loan eligibility:

Java 
public class LoanEligibility {
    public static void main(String[] args) {
        int age = 25;           // Applicant's age
        double income = 50000;  // Annual income
        int creditScore = 720;  // Credit score
        
        // First check: Age requirement
        if (age >= 21 && age <= 65) {
            // Second check: Income requirement
            if (income >= 30000) {
                // Third check: Credit score
                if (creditScore >= 700) {
                    System.out.println("Congratulations! You qualify for a premium loan.");
                } else if (creditScore >= 600) {
                    System.out.println("You qualify for a standard loan.");
                } else {
                    System.out.println("Sorry, your credit score is too low.");
                }
            } else {
                System.out.println("Sorry, income requirement not met.");
                System.out.println("Minimum income: $30,000");
            }
        } else {
            System.out.println("Sorry, age must be between 21 and 65.");
        }
    }
}
Output: Congratulations! You qualify for a premium loan.

Using Logical Operators to Reduce Nesting

Sometimes nested conditions can be simplified using logical operators (&&, ||):

Java - Simplified Version 
// Instead of deeply nested if statements
if (age >= 21 && age <= 65 && income >= 30000 && creditScore >= 700) {
    System.out.println("You qualify for a premium loan.");
}

Common Mistakes and Best Practices

Common Mistakes

  • Using = instead of ==: The single = is assignment, not comparison. if (x = 5) is wrong; use if (x == 5).
  • Forgetting curly braces: Without braces, only the first statement is part of the if block. Always use braces for clarity.
  • Missing break in switch: Forgetting break causes fall-through to subsequent cases.
  • Comparing strings with ==: Use .equals() to compare String values: if (str.equals("hello"))
  • Incorrect condition order: In if...else if chains, more specific conditions should come first.

Best Practices

  • Use meaningful variable names: isEligible is clearer than x for boolean conditions.
  • Keep conditions simple: Complex conditions should be broken into smaller parts or stored in variables.
  • Always include a default case: In switch statements, include default to handle unexpected values.
  • Limit nesting depth: If you have more than 3 levels of nesting, consider refactoring using methods or early returns.
  • Use switch for discrete values: When testing one variable against multiple constant values, prefer switch over long if-else chains.

Summary

  • The if statement executes code only when a condition is true.
  • The if...else statement provides an alternative path when the condition is false.
  • The if...else if...else chain tests multiple conditions sequentially.
  • The switch statement is ideal for testing one variable against multiple discrete values.
  • Always use break in switch cases to prevent fall-through behavior.
  • Nested decisions allow for complex multi-level conditional logic.
  • Use logical operators (&&, ||) to combine conditions and reduce nesting.
  • Always use .equals() to compare String values, not ==.
