Lesson II

Iterative Structures

Master the art of loops to execute code repeatedly, process collections, and automate repetitive tasks efficiently.

Lesson Overview

Iterative structures, commonly known as loops, allow you to execute a block of code multiple times. Instead of writing the same code over and over, loops automate repetition based on conditions or a predetermined count.

Real-World Context: Loops are everywhere in programming. A music player uses loops to repeat songs. Games use loops to update the screen 60 times per second. Search engines loop through millions of web pages to find results. E-commerce sites loop through product lists to display items.

Learning Outcomes

By the end of this lesson, you will be able to:

  • Implement while loops for condition-based iteration
  • Apply do-while loops when at least one execution is required
  • Construct for loops for counter-controlled iteration
  • Design nested loops for multi-dimensional processing
  • Control loop execution using break and continue statements

1. The While Loop

The while loop repeatedly executes a block of code as long as a specified condition remains true. The condition is checked before each iteration.

Syntax

Java Syntax 
while (condition) {
    // Code to execute while condition is true
}

Example: Counting from 1 to 5

Java 
public class WhileExample {
    public static void main(String[] args) {
        int count = 1;  // Initialize counter
        
        // Loop while count is less than or equal to 5
        while (count <= 5) {
            System.out.println("Count: " + count);
            count++;  // Increment counter to avoid infinite loop
        }
        
        System.out.println("Loop finished!");
    }
}
Output:
Count: 1
Count: 2
Count: 3
Count: 4
Count: 5
Loop finished!

Example: Sum of Numbers

Calculate the sum of numbers from 1 to 10:

Java 
public class SumCalculator {
    public static void main(String[] args) {
        int number = 1;   // Starting number
        int sum = 0;      // Accumulator for sum
        
        while (number <= 10) {
            sum += number;  // Add current number to sum
            number++;       // Move to next number
        }
        
        System.out.println("Sum of 1 to 10: " + sum);
    }
}
Infinite Loop Warning: If the condition never becomes false, the loop runs forever. Always ensure your loop has a way to terminate (e.g., incrementing a counter or changing a condition variable).

2. The Do-While Loop

The do-while loop is similar to the while loop, but it checks the condition after executing the code block. This guarantees at least one execution.

Syntax

Java Syntax 
do {
    // Code to execute at least once
} while (condition);

Example: Menu System

A menu that displays at least once and repeats until the user chooses to exit:

Java 
import java.util.Scanner;

public class MenuSystem {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        int choice;
        
        do {
            // Display menu options
            System.out.println("\n=== Main Menu ===");
            System.out.println("1. Start Game");
            System.out.println("2. Load Game");
            System.out.println("3. Settings");
            System.out.println("4. Exit");
            System.out.print("Enter your choice: ");
            
            choice = scanner.nextInt();
            
            // Process user choice
            switch (choice) {
                case 1:
                    System.out.println("Starting new game...");
                    break;
                case 2:
                    System.out.println("Loading saved game...");
                    break;
                case 3:
                    System.out.println("Opening settings...");
                    break;
                case 4:
                    System.out.println("Goodbye!");
                    break;
                default:
                    System.out.println("Invalid choice. Try again.");
            }
        } while (choice != 4);  // Continue until user chooses exit
        
        scanner.close();
    }
}
When to Use: Use do-while when you need the code to run at least once before checking the condition, such as input validation, menu systems, or game loops.

While vs Do-While Comparison

Feature while do-while
Condition check Before loop body After loop body
Minimum executions 0 (may never run) 1 (always runs at least once)
Semicolon after condition No Yes (required)

3. The For Loop

The for loop is ideal when you know exactly how many times you need to iterate. It combines initialization, condition, and increment/decrement into a single line.

Syntax

Java Syntax 
for (initialization; condition; update) {
    // Code to execute
}

Components:

  • Initialization: Executed once before the loop starts (e.g., int i = 0)
  • Condition: Checked before each iteration; loop continues while true
  • Update: Executed after each iteration (e.g., i++)

Example: Basic For Loop

Java 
public class ForLoopExample {
    public static void main(String[] args) {
        // Print numbers 1 through 5
        for (int i = 1; i <= 5; i++) {
            System.out.println("Count: " + i);
        }
    }
}

Example: Countdown

Java 
public class Countdown {
    public static void main(String[] args) {
        System.out.println("Countdown started!");
        
        // Count down from 10 to 1
        for (int i = 10; i >= 1; i--) {
            System.out.println(i);
        }
        
        System.out.println("Liftoff!");
    }
}

Example: Multiplication Table

Java 
public class MultiplicationTable {
    public static void main(String[] args) {
        int number = 7;  // Number for multiplication table
        
        System.out.println("Multiplication Table for " + number);
        System.out.println("========================");
        
        // Generate multiplication table from 1 to 10
        for (int i = 1; i <= 10; i++) {
            int result = number * i;
            System.out.println(number + " x " + i + " = " + result);
        }
    }
}

4. Nested Loops

A nested loop is a loop inside another loop. The inner loop completes all its iterations for each iteration of the outer loop. This is useful for working with multi-dimensional data or creating patterns.

Example: Rectangle Pattern

Java 
public class RectanglePattern {
    public static void main(String[] args) {
        int rows = 4;     // Number of rows
        int cols = 6;     // Number of columns
        
        // Outer loop controls rows
        for (int i = 1; i <= rows; i++) {
            // Inner loop controls columns
            for (int j = 1; j <= cols; j++) {
                System.out.print("* ");
            }
            System.out.println();  // Move to next line
        }
    }
}
Output:
* * * * * *
* * * * * *
* * * * * *
* * * * * *

Example: Right Triangle Pattern

Java 
public class TrianglePattern {
    public static void main(String[] args) {
        int height = 5;  // Triangle height
        
        // Outer loop: each row
        for (int row = 1; row <= height; row++) {
            // Inner loop: print stars equal to current row number
            for (int star = 1; star <= row; star++) {
                System.out.print("*");
            }
            System.out.println();  // New line after each row
        }
    }
}
Output:
*
**
***
****
*****

Example: Multiplication Table Grid

Java 
public class MultiplicationGrid {
    public static void main(String[] args) {
        int size = 5;
        
        // Print header row
        System.out.print("  |");
        for (int i = 1; i <= size; i++) {
            System.out.printf("%4d", i);
        }
        System.out.println();
        System.out.println("--+--------------------");
        
        // Print multiplication table
        for (int row = 1; row <= size; row++) {
            System.out.printf("%2d|", row);  // Row header
            for (int col = 1; col <= size; col++) {
                System.out.printf("%4d", row * col);
            }
            System.out.println();
        }
    }
}

5. The Break Statement

The break statement immediately terminates the current loop and transfers control to the statement following the loop. It's useful for exiting a loop when a specific condition is met.

Example: Finding a Number

Java 
public class FindNumber {
    public static void main(String[] args) {
        int[] numbers = {10, 25, 33, 47, 52, 68, 71};
        int target = 47;
        boolean found = false;
        
        for (int i = 0; i < numbers.length; i++) {
            System.out.println("Checking index " + i + ": " + numbers[i]);
            
            if (numbers[i] == target) {
                System.out.println("Found " + target + " at index " + i);
                found = true;
                break;  // Exit loop immediately
            }
        }
        
        if (!found) {
            System.out.println(target + " not found in the array.");
        }
    }
}
Output:
Checking index 0: 10
Checking index 1: 25
Checking index 2: 33
Checking index 3: 47
Found 47 at index 3

6. The Continue Statement

The continue statement skips the rest of the current iteration and moves to the next iteration of the loop. It's useful for skipping specific values while processing.

Example: Skip Odd Numbers

Java 
public class SkipOddNumbers {
    public static void main(String[] args) {
        System.out.println("Even numbers from 1 to 10:");
        
        for (int i = 1; i <= 10; i++) {
            // Skip odd numbers
            if (i % 2 != 0) {
                continue;  // Skip to next iteration
            }
            System.out.println(i);
        }
    }
}
Output:
Even numbers from 1 to 10:
2
4
6
8
10

Break vs Continue Comparison

Statement Effect Use Case
break Exits the entire loop Stop searching when target found
continue Skips to next iteration Skip invalid or unwanted values

Common Mistakes and Best Practices

Common Mistakes

  • Infinite loops: Forgetting to update the loop variable causes the loop to run forever.
  • Off-by-one errors: Using < instead of <= (or vice versa) can cause loops to iterate one too many or too few times.
  • Wrong loop type: Using while when for is clearer, or do-while when minimum execution isn't needed.
  • Modifying loop variable inside loop: Changing the loop counter inside the loop body can cause unexpected behavior.
  • Missing semicolon in do-while: The do-while loop requires a semicolon after the condition.

Best Practices

  • Use descriptive counter names: Use row, col, student instead of just i, j, k when appropriate.
  • Keep loops simple: Each loop should do one thing well. Complex logic should be moved to methods.
  • Use for-each for collections: When iterating through arrays or collections without needing the index, use the enhanced for loop.
  • Limit nesting depth: Deeply nested loops are hard to read. Consider refactoring if you have more than 3 levels.
  • Use break sparingly: Overuse of break can make code harder to follow. Consider restructuring the loop condition.

for (int i = 0; i < 5; i++) { }

3. What does the continue statement do?

4. In a nested loop, how many total iterations occur if the outer loop runs 3 times and the inner loop runs 4 times?

Summary

  • The while loop checks the condition before each iteration and may not execute at all.
  • The do-while loop checks the condition after each iteration, guaranteeing at least one execution.
  • The for loop combines initialization, condition, and update in one line—ideal for counter-controlled loops.
  • Nested loops allow processing of multi-dimensional data; the inner loop completes fully for each outer iteration.
  • The break statement exits the loop immediately.
  • The continue statement skips the current iteration and moves to the next.
  • Always ensure loops have a termination condition to avoid infinite loops.
