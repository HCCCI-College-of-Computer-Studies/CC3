Lesson VIII

Regular Expressions

Master pattern matching and text manipulation using regular expressions for validation and parsing.

← Previous: Accessing Files
Next: Essential Java Classes →

Lesson Overview

Regular expressions (regex) are patterns used to match character combinations in strings. They're powerful tools for searching, validating, and manipulating text data.

Real-World Context: Regex is used to validate email addresses, phone numbers, passwords, and credit card numbers. It's also used for parsing log files, extracting data from text, and search-and-replace operations.

Learning Outcomes

At the end of this lesson, the student should be able to:

  • Identify standard Regex meta-characters and their functions in pattern matching.
  • Analyze complex Regex strings to predict which input patterns will be validated or rejected.
  • Develop a robust input-validation module that filters user data against specific organizational rules.

1. Regular Expression Basics

Common Metacharacters

Pattern Description Example
.Any single charactera.c matches "abc", "a1c"
\dAny digit (0-9)\d\d matches "42"
\wWord character (a-z, A-Z, 0-9, _)\w+ matches "hello"
\sWhitespacea\sb matches "a b"
*Zero or moreab* matches "a", "ab", "abb"
+One or moreab+ matches "ab", "abb"
?Zero or onecolou?r matches "color", "colour"
{n}Exactly n times\d{4} matches "2024"
[abc]Character class[aeiou] matches any vowel
^Start of string^Hello matches "Hello world"
$End of stringworld$ matches "Hello world"

Using Regex in Java

Java 
import java.util.regex.*;

public class RegexBasics {
    public static void main(String[] args) {
        // Simple matching with String.matches()
        String text = "Hello123";
        
        // Check if contains only letters
        System.out.println(text.matches("[a-zA-Z]+"));  // false
        
        // Check if contains letters and digits
        System.out.println(text.matches("[a-zA-Z0-9]+"));  // true
        
        // Using Pattern and Matcher
        Pattern pattern = Pattern.compile("\\d+");  // Find digits
        Matcher matcher = pattern.matcher("Order 123 has 5 items");
        
        while (matcher.find()) {
            System.out.println("Found: " + matcher.group());
        }
        // Output: Found: 123, Found: 5
    }
}

2. Validation Examples

Java 
public class Validation {
    
    // Email validation
    public static boolean isValidEmail(String email) {
        String regex = "^[\\w.-]+@[\\w.-]+\\.[a-zA-Z]{2,}$";
        return email.matches(regex);
    }
    
    // Phone number (PH format: 09XX-XXX-XXXX)
    public static boolean isValidPhone(String phone) {
        String regex = "^09\\d{2}-\\d{3}-\\d{4}$";
        return phone.matches(regex);
    }
    
    // Password (min 8 chars, 1 upper, 1 lower, 1 digit)
    public static boolean isStrongPassword(String password) {
        if (password.length() < 8) return false;
        boolean hasUpper = password.matches(".*[A-Z].*");
        boolean hasLower = password.matches(".*[a-z].*");
        boolean hasDigit = password.matches(".*\\d.*");
        return hasUpper && hasLower && hasDigit;
    }
    
    public static void main(String[] args) {
        System.out.println(isValidEmail("user@example.com"));  // true
        System.out.println(isValidPhone("0917-123-4567"));   // true
        System.out.println(isStrongPassword("Pass123!"));    // true
    }
}

Summary

  • Regular expressions are patterns for matching text.
  • Use \d for digits, \w for word characters, \s for whitespace.
  • Quantifiers: * (0+), + (1+), ? (0 or 1), {n} (exactly n).
  • Java uses String.matches() for simple matching, Pattern and Matcher for complex operations.
  • Escape backslashes in Java strings: \\d instead of \d.
